# Requirement Alignment

This folder maps directly to `Option 2: Payment Appchain Built on OP Stack`.

## What 01 covers

- Deploy a basic OP Stack appchain
- Keep Sepolia as the L1 testnet
- Run the sequencer, batcher, and proposer locally
- Produce the chain environment needed by the payment contract, frontend, and benchmark work

## What 01 does not cover

- Payment contract implementation
- Merchant records logic
- Frontend pages
- TPS and fee analysis scripts

Those belong to:

- `02-smart-contract`
- `03-backend-simulation`
- `04-frontend`
- `05-pm-benchmark`
